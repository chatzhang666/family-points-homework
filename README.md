# 快乐任务屋

供家长和孩子在家庭 Wi-Fi 内使用的作业积分网页应用。孩子在 iPhone 11 上把网页添加到主屏幕后，可以查看今日任务、提交完成、查看积分和申请奖品；家长负责发布作业、设置每日任务及挑战、审核发分和管理兑换。

## 首次使用

1. 在 QNAP `TS-462C` 的 App Center 安装 `Container Station`。
2. 将本项目文件夹放到 NAS 可管理的目录，在 Container Station 中用 `docker-compose.yml` 建立应用。
3. 容器启动后，在家庭 Wi-Fi 中访问 `http://<NAS局域网地址>:3080`。
4. 第一次打开时创建家长账号；家长首页会显示一次性邀请码。
5. 孩子在 iPhone Safari 打开同一地址，选择“孩子加入”，输入邀请码和自己的数字密码。
6. 在 iPhone Safari 的分享菜单中选择“添加到主屏幕”，以后从“快乐任务屋”图标进入；登录有效期内会自动进入孩子端或家长端。

## 在开发电脑上运行

```bash
npm install
npm run dev
```

打开 `http://localhost:3000`。数据库自动保存在 `data/family-points.db`。

## 功能说明

- 学校作业：家长粘贴微信群文字，系统按行拆为候选任务，确认发布时默认每项 `1` 分且可修改。
- 每日任务：首次创建家庭时提供带 `30` 分钟计时的“阅读 30 分钟”和“整理书包”，默认 `2` 分；家长可修改、删除或新增。每天凌晨 `2:00` 切换新一天，未完成任务不顺延。
- 全部完成奖励：家长可设置当天全部任务完成后的额外积分；孩子提交当天最后一项任务时看到专属庆祝，家长确认任务全部完成后奖励到账。
- 高分挑战：可选择整体完成一次发分，或设置多个步骤逐项审批发分；可设置单次或每日出现。
- 计时任务：家长可为普通任务或整体挑战设置分钟数；孩子开始计时后可看到倒计时和进度条，到时才可提交。
- 积分安全：孩子提交后会看到庆祝反馈，但仅家长批准才记入积分；记录页只保留积分增减数值与时间。
- 心愿商店：孩子提交兑换申请时立即扣除积分，家长拒绝申请时返还积分。

## NAS 更新

推荐的更新方式是固定使用稳定标签 `nas-stable`。以后每次更新只需要替换容器，不需要再导入 tar：

1. 在这台 Mac 运行：

```bash
npm run nas:publish
```

2. 在 QNAP `Container Station` 中提取镜像：
   `192.168.31.98:5500/family-points:nas-stable`
3. 停止并删除旧的 `family-points` 容器，但不要删除数据目录 `/Public/family-points/data`。
4. 用 [docker-compose.nas.yml](/Users/zhanglintong/Documents/积分学习/docker-compose.nas.yml) 的同样配置重新创建 `family-points` 容器：
   - 镜像：`192.168.31.98:5500/family-points:nas-stable`
   - 端口：`32769:3000`
   - 数据目录：`/Public/family-points/data:/app/data`

这样以后不需要再手工改版本号，也不需要重新走离线导入流程。数据库放在 NAS 持久目录里，替换容器不会丢数据。

## NAS 数据与备份

- NAS 上的数据目录固定为 `/Public/family-points/data`，更新或重建容器时继续挂载这个目录即可保留记录。
- 家长端“记录”页可下载 JSON 明细备份，方便查看任务、积分和奖品记录。
- 可恢复的完整备份文件为 `data/family-points.db`。请使用 QNAP 定期备份该目录；恢复时停止容器，以备份的数据库文件替换该文件后重新启动容器。
- 当前 NAS 访问地址为 `http://<NAS局域网地址>:32769`。仅供局域网使用，请勿将该端口开放至公网。

## 技术结构

- `Next.js` 与 `TypeScript`：响应式家长端和孩子端。
- `SQLite` 与 `Drizzle ORM`：家庭数据持久化和结构定义。
- `Docker Compose`：部署到支持 Container Station 的 QNAP NAS。

卡通界面中的女孩角色以家庭提供的参考形象生成，用作积分卡片、商店和庆祝反馈中的陪伴角色；页面不使用参考图中的动画狗角色素材。
